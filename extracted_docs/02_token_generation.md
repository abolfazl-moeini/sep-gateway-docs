# بخش ۲: بسترسازی سمت فروشنده و دریافت توکن (صفحات ۶ تا ۱۰)

این بخش شامل مراحل پیاده‌سازی تراکنش در سمت فروشنده و راهنمای کامل دریافت توکن پرداخت (Token Request) به همراه پارامترهای فنی مربوطه است.

---

## ۲.۱. انجام تراکنش (Transaction Process)

فروشنده خریدار را به نقطه‌ای می‌رساند که آماده دریافت پول و نهایی کردن خرید است (مثلاً در انتهای سبد خرید، خرید نهایی بلیط و غیره). در این نقطه فروشنده باید مراحل زیر را دنبال نماید:

1. **دریافت توکن:** فروشنده جهت هدایت خریدار به صفحه پرداخت نیاز به **توکن (Token)** دارد که یک کد منحصر به فرد برای هر تراکنش است.
2. **ارسال اطلاعات:** توکن از طریق ارسال پارامترهای تراکنش به آدرس صفحه پرداخت دریافت می‌شود.
3. **هدایت کاربر:** پس از دریافت توکن، پذیرنده کاربر خود را با کد توکن به صفحه پرداخت سامان هدایت می‌کند.

> [!IMPORTANT]
> **ثبت IP سرور:** جهت دریافت توکن، باید IP سرور پذیرنده در مشخصات پذیرنده نزد شرکت پرداخت الکترونیک سامان ثبت شده باشد.
> 
> **نوع درخواست:** ارسال اطلاعات برای دریافت توکن باید به صورت یک درخواست HTTP و متد **POST** باشد.
> 
> **فرمت پاسخ:** اطلاعاتی که از سمت سرور پرداخت الکترونیک سامان به پذیرنده برگشت داده می‌شود با فرمت **JSON** است.
> 
> **اعتبار توکن:** هر توکن ایجاد شده به صورت پیش‌فرض تا **۲۰ دقیقه** اعتبار دارد. پذیرنده می‌تواند این مقدار را با ارسال پارامتر `TokenExpiryInMin` تغییر دهد. در صورتی که بعد از زمان انقضای توکن خریدار به درگاه فرستاده شود، روند پرداخت متوقف خواهد شد.

---

## ۲.۲. پارامترهای ارسالی جهت دریافت توکن (Token Request Payload)

جهت دریافت توکن، باید پارامترهای زیر را به آدرس وب‌سرویس توکن ارسال نمایید:

| نام پارامتر | نوع داده | وضعیت الزامی | توضیحات |
| :--- | :---: | :---: | :--- |
| `Action` | String | **اجباری** | مشخص‌کننده نوع عملیات درخواستی است. برای دریافت توکن مقدار آن باید دقیقاً `"token"` باشد (به حروف بزرگ و کوچک حساس نیست). |
| `TerminalId` | String | **اجباری** | شماره ترمینال فروشنده که از سوی شرکت پرداخت الکترونیک سامان کیش اعلام می‌شود. |
| `Amount` | Integer | **اجباری** | مبلغ خرید به ریال. باید عدد صحیح (Integer) بدون اعشار و کاراکتر اضافی باشد (در استفاده از سیستم‌های فروشگاهی آماده خارجی باید به این مورد توجه شود). |
| `ResNum` | String | **اجباری** | شماره یکتای تراکنش خرید که از طرف فروشنده تولید می‌شود تا از پرداخت تکراری جلوگیری شده و قابلیت استعلام داشته باشد. |
| `RedirectURL` | String | **اجباری** | آدرس صفحه بازگشت پذیرنده که خریدار پس از پرداخت به آن هدایت می‌شود. |
| `CellNumber` | String | اختیاری | شماره موبایل خریدار جهت بازیابی یا ذخیره‌سازی اطلاعات کارت‌های وی در درگاه. |
| `TokenExpiryInMin` | Integer | اختیاری | مدت زمان اعتبار توکن برحسب دقیقه (کمینه ۲۰ و بیشینه ۳۶۰۰ دقیقه). در صورت عدم ارسال، مقدار پیش‌فرض ۲۰ دقیقه لحاظ می‌شود. در صورت ارسال مقدار خارج از این بازه، مقادیر مرزی (کمینه یا بیشینه) در نظر گرفته خواهند شد. |
| `Wage` | Integer | اختیاری | مبلغ کارمزد تراکنش. برای پذیرندگانی که مایل به تسویه از روش تسهیم هستند. مبلغ کسر شده نهایی از کارت مجموع `Amount` و `Wage` خواهد بود. |
| `AffectiveAmount` | Integer | اختیاری | مبلغ کسر شده از کارت خریدار؛ ویژه پذیرندگانی که از سامانه تخفیف استفاده می‌کنند. |
| `HashedCardNumber` | String | اختیاری | شماره کارت هش شده با الگوریتم غیرقابل برگشت MD5. در صورتی که پذیرنده بخواهد خریدار را ملزم به استفاده از کارت‌های خاصی نماید، هش MD5 شماره کارت‌ها را با کاراکترهای جداکننده `\|` یا `;` یا `,` ارسال می‌کند. (حداکثر ۱۰ عدد کارت). |

## ۲.۳. نمونه‌های درخواست و پاسخ (Request & Response Examples)

### الف) نمونه درخواست ارسالی برای دریافت توکن (Request JSON)
* **آدرس وب‌سرویس:** `POST https://sep.shaparak.ir/onlinepg/onlinepg`

```json
{
  "action": "token",
  "TerminalId": "0000",
  "Amount": 12000,
  "ResNum": "1qaz@WSX",
  "RedirectUrl": "http://mysite.com/receipt",
  "CellNumber": "9120000000"
}
```

### ب) نمونه پاسخ موفق دریافت توکن (Successful Response JSON)
```json
{
  "status": 1,
  "token": "2c3c1fefac5a48geb9f9be7e445dd9b2"
}
```

### ج) نمونه پاسخ خطا در دریافت توکن (Error Response JSON)
```json
{
  "status": -1,
  "errorCode": "5",
  "errorDesc": "پارامترهای ارسال شده نامعتبر است.; شماره تراکنش پذیرنده الزامی است"
}
```
