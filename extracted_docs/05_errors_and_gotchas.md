# کدهای خطا و نکات حیاتی

## خطاهای رایج FAQ

| خطا | علت / اقدام |
| :--- | :--- |
| `-33` دسترسی به نسخه درگاه | باید `onlinepg` باشد؛ نه LightIPG/MobilePG |
| آدرس ارجاع‌دهنده معتبر نیست | دامنه/`www`/https با ثبت SEP یکی باشد |
| کد پذیرنده نامعتبر | MID به‌جای TerminalId |
| IP نامعتبر | توکن: کد ۸؛ Verify: `-106` (FAQ گاهی `-18`) — IP را به SEP اعلام کنید |
| `03` هنگام تراکنش | ترمینال غیرفعال یا شبا تسهیم اشتباه؛ `IR` حروف بزرگ |
| `-111` در Verify | ساختار Body اشتباه (`RefNum` / `TerminalNumber`) |
| امکان تایید / `-6` | Verify بعد از ۳۰ دقیقه |
| Session Is Null | Refresh Callback یا تغییر شبکه کاربر |
| 504 / Timeout | پورت ۴۴۳ و مسیر به `91.240.182.20`؛ سرور خارج ایران ممنوع |
| مبلغ بیش از حد مجاز | محدودیت ترمینال (مدارک ناقص) |
| پایانه مجوز ندارد | مبلغ < ۲۰۰۰ ریال |

## نکات پیاده‌سازی

1. **Double Spending:** SEP مصرف مجدد `RefNum` را رد نمی‌کند؛ در DB خودتان یکتا نگه دارید.
2. **مبلغ:** همیشه `OrginalAmount` را با سفارش مقایسه کنید.
3. **Retry:** روی Timeout شبکه Retry کنید؛ فقط با کد منفی صریح ناموفق بدانید.
4. **هش:** ورودی توکن = MD5؛ خروجی Callback = SHA256؛ کارت خام ذخیره نشود.
5. **Referrer:** Redirect فقط از دامنه ثبت‌شده.
6. **تسهیم:** شبا دقیقاً تاییدشده + `IR` بزرگ.
7. **املای Verify:** `OrginalAmount`, `StraceDate`, `StraceNo`.
